# 컬리 라벨 생성 기능 스펙 (v1)

## 1. 기능명
- **컬리 라벨 생성**

## 2. 목표
- 엑셀 업로드 기반으로 컬리 입고 라벨을 생성한다.
- 오출고 방지를 위해 헤더/데이터 검증을 엄격하게 적용한다.
- 라벨의 `상품코드` 값은 반드시 엑셀의 `마스터코드`에서만 매핑한다.

## 3. 입력 파일 정책
- 지원 확장자: `.xlsx`, `.xls` (v1)
- 처리 시트: 첫 번째 시트 (v1)
- 헤더는 1행 기준으로 읽는다.

## 4. 필수 헤더 (정확 일치)
다음 7개 헤더가 **정확히** 존재해야 한다.

1. `발주코드`
2. `상품명`
3. `마스터코드`
4. `유통기한/소비기한`
5. `박스당입수`
6. `발주확정 수량(낱개)`
7. `발주확정 수량(박스)`

> 호환 허용(제한적): 기존 파일 포맷 대응을 위해 아래 3개는 대체 헤더를 허용한다.  
> - `유통기한/소비기한` ↔ `유통기한`  
> - `발주확정 수량(낱개)` ↔ `총입고수량(낱개)`  
> - `발주확정 수량(박스)` ↔ `전체박스수`

### 4-1. 엄격 규칙
- 문서에 명시한 호환 대체 헤더 외 동의어/유사어는 허용 안 함.
- 예: `상품코드`, `발주 코드`, `총입고수량`(낱개 미포함) 등은 모두 불허.
- 공백/철자/괄호 차이도 불일치로 처리한다.
- 헤더 1개라도 누락/불일치이면 전체 생성 중단.

## 5. 필드 매핑 규칙
- `라벨.발주코드 = 엑셀.발주코드`
- `라벨.상품명 = 엑셀.상품명`
- `라벨.상품코드 = 엑셀.마스터코드` (**고정 규칙**)
- `라벨.유통기한 = 엑셀.유통기한/소비기한` (`(유)` 접두어 제거)
- `라벨.박스내입수량 = 엑셀.박스당입수`
- `라벨.총입고수량 = 엑셀.발주확정 수량(낱개)`
- `라벨.전체박스수 = 엑셀.발주확정 수량(박스)`

## 6. 라벨 생성 규칙
- 한 행(row) 당 `전체박스수 = N`이면 라벨 N장을 생성한다.
- 각 라벨은 아래 값을 가진다.
  - `boxNo`: 1..N
  - `totalBoxes`: N
- 즉, 박스 1개 = 라벨 1장.

## 7. 데이터 검증 규칙
각 행마다 아래를 검증한다.

### 7-1. 필수값
- `발주코드`, `상품명`, `마스터코드`, `유통기한/소비기한`은 빈 값 불가.

### 7-2. 숫자값
- `박스당입수`, `발주확정 수량(낱개)`, `발주확정 수량(박스)`는 숫자여야 한다.
- `발주확정 수량(박스)`는 양의 정수여야 한다.
- `박스당입수`, `발주확정 수량(낱개)`는 0 이상 숫자 허용(운영정책에 따라 1 이상으로 강화 가능).

### 7-3. 실패 정책
- 오류가 1건이라도 있으면 전체 생성 중단(Fail Fast).
- 부분 성공/부분 출력 금지.
- 오류는 `행번호 + 사유` 형태로 사용자에게 제공.

## 8. 에러 메시지 가이드

### 8-1. 헤더 오류
- `필수 헤더 누락: 마스터코드`
- `허용되지 않은 헤더명: 상품코드 (마스터코드만 허용)`

### 8-2. 행 데이터 오류
- `12행: 발주확정 수량(박스)은 양의 정수여야 합니다.`
- `18행: 마스터코드 값이 비어 있습니다.`

### 8-3. 공통
- `오류를 수정한 뒤 다시 업로드해주세요. (라벨 미생성)`

## 9. 출력 포맷(라벨 1장)
- 발주코드
- 공급사명(고정값, 별도 설정 시 적용)
- 상품명
- 상품코드(=마스터코드)
- 유통기한
- 수량/총수량: `박스 내 입수량 (X) / 총 입고수량 (Y)`
- C/T: `박스 번호 (i) / 전체 박스 수 (N)`

## 10. 구현 순서 (권장)
1. 업로드 파서 작성 (엑셀 → row array)
2. 헤더 정확 일치 검증 모듈 작성
3. 행 단위 데이터 검증 모듈 작성
4. 라벨 아이템 빌더 작성 (`row -> labels[]`)
5. 미리보기 렌더/다운로드(PDF) 연동
6. 실패 메시지/UX 정리

## 11. 테스트 체크리스트
- 정상 헤더 + 정상 데이터 → 라벨 생성 성공
- `마스터코드` 대신 `상품코드` 사용 → 즉시 실패
- `전체박스수` 소수/문자 → 실패
- 필수값 빈 값 포함 → 실패
- 오류 1건 존재 시 전체 미출력 확인
